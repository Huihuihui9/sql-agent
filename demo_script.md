# 4 分钟演示脚本 — SQL 数据分析 Agent

## 概览

| 项目 | 说明 |
|------|------|
| 仓库 | github.com/Huihuihui9/sql-agent |
| 技术栈 | Python + LangChain + DeepSeek + SQLite + Streamlit + Docker |
| 核心能力 | 自然语言 → SQL → 安全执行 → 结果展示 |

---

## 脚本正文（4 分钟读完）

### 0:00 - 开场（15 秒）

> "大家好，今天演示的是一个 **SQL 数据分析 Agent**。
> 它的核心能力是：你说人话，AI 帮你写 SQL 查数据库。"

---

### 0:15 - 项目背景（30 秒）

> "很多业务人员想查数据但不会写 SQL，每次都找开发帮忙。
> 这个项目解决的就是这个问题——用自然语言直接查数据库。
>
> 技术架构分三层：
> - **LLM 层**：DeepSeek API，负责理解问题、生成 SQL、解释结果
> - **安全层**：只允许 SELECT、白名单表、危险 SQL 拦截
> - **界面层**：Streamlit Web 界面"

---

### 0:45 - 功能演示 1：自然语言查数据库（90 秒）

> "我们来实际操作一下。
>
> 数据库有三张表：用户、产品、订单。
>
> 我问第一个问题：**'哪个产品卖得最多？'**
>
> [演示：输入问题 → 自动生成 SQL → 显示结果]
>
> 系统会自动生成带 JOIN 和 GROUP BY 的 SQL 查询，
> 返回 'Python编程书'，销量 5 本。
>
> 再试一个复杂的问题：**'北京的用户花了多少钱？'**
>
> [演示：输入问题 → 带 WHERE 条件的 SQL → 金额汇总]
>
> 系统会理解 '北京' 是城市维度，'花了多少钱' 是金额汇总，
> 自动关联 users 和 orders 两张表。"

---

### 2:15 - 功能演示 2：Agent 多工具协作（60 秒）

> "这个项目的另一个亮点是 **Agent 多工具协作**。
>
> 系统不止能查数据库，它还集成了三个工具：
> - **SQL 查询**：查数据库
> - **计算器**：做数学计算
> - **网页搜索**：获取实时信息
>
> 比如问：**'计算 15800 打 85 折是多少'**
> Agent 会自动调用计算器工具，返回 13430。
>
> 或者问：**'Python 是谁创造的？'**
> Agent 会自动搜索 Wikipedia，返回 Guido van Rossum 的信息。
>
> Agent 的核心思想是：**LLM 自己决定下一步要调用什么工具**，
> 而不是走固定的流程。"

---

### 3:15 - 安全设计（30 秒）

> "作为一个数据库工具，安全是重中之重。
> 我做了三层防护：
> 1. **只允许 SELECT**：DROP、DELETE 等危险操作直接拦截
> 2. **白名单表**：只能查 users、products、orders 三张表
> 3. **行数限制**：最多返回 30 行，防止内存溢出"

---

### 3:45 - 部署 + 总结（15 秒）

> "项目已用 Docker 容器化，支持一键部署。
>
> 完整的演示可以在 GitHub 上找到：
> **github.com/Huihuihui9/sql-agent**
>
> 谢谢！"
